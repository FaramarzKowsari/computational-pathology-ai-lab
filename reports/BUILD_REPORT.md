# Build and Delivery Report

## Delivered system

The repository is a production-oriented computational-pathology research platform that combines:

1. Medical AI research methodology;
2. machine-learning engineering and MLOps;
3. computer-vision engineering;
4. academic teaching and technical authorship.

## Implemented, executable components

- Python package under `src/cpathlab`;
- synthetic patch generator and data validator;
- group-aware and stratified split logic;
- preprocessing, augmentation, and educational Reinhard-style normalization;
- baseline CNN, transfer-learning factory, tiny ViT, foundation-model adapter, and attention MIL;
- training, inference, metrics, calibration, confidence intervals, error analysis, domain-shift transforms, explainability, uncertainty, and drift utilities;
- FastAPI service and optional Streamlit UI;
- Docker, Docker Compose, DVC, MLflow hooks, model-registry structure, Kubernetes examples, CI, Pages deployment, tests, linting, and typing;
- Model Card, Data Card, ethics, methodology, reproducibility, benchmark, teaching, portfolio, release, SEO, Search Console, and Zenodo documentation;
- SEO-ready website, JSON-LD, CodeMeta, Zenodo metadata, social preview, sitemap, and robots file;
- companion-book metadata and verified placeholder PDF.

## External execution required

The following require licensed data, external services, authorized weights, or suitable compute:

- real Histopathologic Cancer Detection training and evaluation;
- pathology foundation-model downloads and benchmarks;
- WSI workflows requiring OpenSlide-compatible data;
- GPU profiling and mixed-precision experiments;
- MLflow/DVC remote infrastructure;
- external and clinical validation;
- final infographic book production.

## Scientific integrity

No real-data AUC, accuracy, foundation-model result, clinical claim, or medical-device claim was fabricated. Benchmark templates use `Not Yet Benchmarked` until evidence files exist.

## Preservation of the original assignment

The exact binary/text originals are intentionally migrated from the user's existing local clone. The delivery does not invent substitutes. Run `PREPARE_WINDOWS.bat` after copying the new files over the existing repository. The migration preserves the original notebooks, Kaggle submission, and leaderboard image under `legacy/cu_boulder_original_assignment/`.
