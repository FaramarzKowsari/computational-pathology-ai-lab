# Contributing

1. Open an issue describing the proposed scientific or engineering change.
2. Create a focused branch and include tests.
3. Run `make quality` and `make test`.
4. Document data provenance, split logic, randomness, and compute requirements.
5. Never commit private health information, restricted datasets, access tokens, or fabricated metrics.
6. Mark unexecuted experiments as `Not Yet Benchmarked`.

Medical-AI contributions must include a limitations note and must not be presented as clinical advice.
